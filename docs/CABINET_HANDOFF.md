## Три статичні сторінки (public/, авто-масштаб стейдж 1920×1080, transform: scale)
1. **`khotyn_slide.html`** — слайд/вхід. Клік на `.wb-title` → email/пароль → `POST /api/slide-login` → `REDIRECT_TO = '/kabinet.html'`.
2. **`kabinet.html`** — проміжний кабінет (усі ролі). Шапка KPI = вся лікарня; head_dept клікає назву свого відділення → `/head-cabinet.html?dept=…`.
3. **`head-cabinet.html`** — кабінет завідувача (відкривати з `?dept=<точна назва БД>`).

**ВАЖЛИВО — нюанс Vercel:** статичний `public/X.html` НЕ можна називати як маршрут `pages/X.js` (сторінка «забирає» імʼя, `/X.html` → 404 на Vercel, локально не відтворюється). Тому `cabinet.html` → `kabinet.html` (бо є `pages/cabinet.js`). Захист статичних кабінетів — на клієнті: `fetch('/api/me')` 401 → `location.replace('/khotyn_slide.html')`.

## kabinet.html (поточний стан)
- Дефолт рік = **поточний − 1** (динамічно, `new Date().getFullYear()-1`), пігулка активна одразу.
- Профіль (`.me-bar`): весь ПІБ одним рядком; кнопка **«Вийти» перенесена на смугу** «Чергові лікарі» ліворуч (`.work-band .me-logout`, left 134).
- **Розгортка KPI відділень при наведенні** (замість старої drawer-шторки): під назвою відділення зʼявляються 4 показники — **Випадків / Пацієнтів / Лікарів / Ліжок** + рядок «Завідувач: ПІБ». Дані: `deptProfileYear` (випадків, унікальних) + `deptHead` (doctors, beds).
  - Терапевтичний список (`.dept-list`): розгортка ПІД назвою, нижня частина (смуга+поля логіну) зсувається вниз (`body.expand-pushed`, CSS-var `--push`).
  - Хірургічний список (`.dept-list2`): розгортка overlay НАД назвою, зсувається лише смуга вгору (`body.expand-pushed-up`).
- **Вертикальна подвійна лінія:** статичні сегменти `.vseg/.vseg2` мають розрив у зоні смуги; при зсуві смуги (`expand-pushed/-up`) суцільні `.vline/.vline2` проявляються (`opacity:1; z-index:3`) і закривають розрив.

## head-cabinet.html (поточний стан — основна робота сесії)
- **Пігулки років:** «ВСІ РОКИ» ПРИБРАНО — рік завжди конкретний (кабінет працює по датах). Дефолт: рік−1 + **поточний місяць** (тобто «сьогодні рік тому», напр. 18.06.2025).
- **Назва відділення** (`.dept-title`) — праворуч угорі (`right:54; top:262`), НЕ клікабельна.
- **Профіль завідувача** (`.me-bar`) — правий нижній кут (`right:54; top:887`): ПІБ одним рядком + посада під ним, по правому краю.
- **Ординаторська** (`.dept-docs-pills`, `left:40; top:250`): єдиний **скрольований список** (max-height 215 ≈ 7 рядків, прихований скролбар, розмиті краї mask), **повне ПІБ** (`deptDocs2.full_name`), **завідувача прибрано** (він у профілі), `z-index:1` (прокрутка під смугою). `.dept-docs-pills2` більше не використовується.
- **Графік по днях** (`.chart-therap`, `periodDaily`) — клік на точку → блок перебуваючих.
- **Блок «перебуває в відділенні»** (`.admissions-box`, `left:600; right:54; top:595`) — хворі, що **перебувають у відділенні на обрану дату** (клік на точку графіка): `admission_date ≤ дата AND (discharge_date ≥ дата OR NULL)`.
  - Рядок: ПІБ · вік · стать · діагноз [· повторних: N] (один рядок); праворуч **смужка діб** (сегмент = 1 доба, `length_of_stay`), синій сегмент = обрана доба, число днів після смужки.
  - Заголовок: «{день тижня} ДД.ММ.РРРР · **перебуває в відділенні** · N · **поступило: P** · **виписано: V**». Усі три — клікабельні фільтри (active=синій, повторний клік знімає). «перебуває» = скидання на всіх.
  - Список — скрол з розмитими краями, низ на рівні кінця вертикальної лінії; сортування за ПІБ А→Я. Авто-показ при завантаженні `showAdmissions(new Date().getDate())`.
- **Взаємодія пацієнт↔лікар:** наведення на пацієнта — лише пацієнт синіє (CSS :hover). **Клік** на пацієнта → підсвічує його лікаря (`.hl`) + **інерційна прокрутка** ординаторської до лікаря по центру (`inertialScrollToCenter`, easeOutCubic). Клік на лікаря → `.doc-active` + показ лише його пацієнтів. Зіставлення по `doc_name` (звірено: збігається для штатних; консультанти інших відділень не в списку). Скидання знімає `.hl`/`.doc-active`.
- **Donut блоків МКХ** (`.dept-pie`, `left:0; top:590`) — топ-5 **блоків МКХ (2-й ступінь ієрархії)** + 6-й сегмент «Інші». Підписи (назва+%) **по зовнішньому колу** з лініями-виносками. R=92, SW=38. Дані: `deptIcdBlocksYear`.

## API ключі (pages/api/stats.js)
- **PUBLIC_KEYS** (без логіну, агреговані без ПІБ): `ovKpiYear, doctorCount(Year), therapeutic/surgicalKpiYear, deptProfile(Year), deptHead, therapeutic/surgicalMonthly, hospitalMonthly, allYears, therapeutic/surgicalTrend, deptOrdinators, deptDocs2, deptDaily, deptIcdPie(Year), deptIcdBlocksYear, periodKpi, periodDaily, periodFlow`.
- **НЕ публічний** (містить ПІБ, лише авторизовані): **`periodAdmissions`** (param `відділення|рік|місяць|день`) — ПІБ (join `patients_best`), вік, стать, діагноз, днів, лікар (`doc_name`), поступив, повторні (`(SELECT COUNT FROM lsmd WHERE patient_id=…)-1`), виписаний. Фільтр = перебуває станом на `make_date(рік,міс,день)`.
- `periodFlow` (param `відділення|рік|місяць|день`) → `{поступило, виписано}` за конкретну дату.
- `deptIcdBlocksYear` (param `назва|рік`) → топ-5 блоків МКХ + «Інші». Блок = CASE по `LEFT(icd_primary,3)` діапазонах (G/I/M); fallback `category_level1`. **Мапінг заточений під неврологію — для хірургічних відділень покаже глави; розширити (S/T/N/K) за потреби.**
- `deptDocs2` — додано `full_name` (COALESCE з emp_name), сортування за full_name.

## Нюанси БД
- `empl.age`/`lsmd.age` — text зі сміттям: `AVG(age::numeric) FILTER (WHERE age ~ '^[0-9]+$')`.
- `icd_10`: `code/category_level1` (глава), `code/diagnosis_level2` (рубрика — **частково NULL!**), `code/diagnosis_level3`. Блоків МКХ у таблиці НЕМАЄ — рахуються через діапазони `LEFT(icd_primary,3)`.
- `lsmd.doc_name` ↔ `lsmd_doctors.doc_name` ↔ `deptDocs2` — збігаються для штатних; сміття: битий юнікод (`Дубе?ць`), дублі (`Сав'юк Т. В. (2)`).
- `app_users`: auth_user_id → role + empl_name_id. Ролі: admin, head_dept, doctor, viewer.
- Госпіталізацій ≈ 110 207, Пацієнтів (distinct) ≈ 67 856.
