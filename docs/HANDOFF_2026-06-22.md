## Технічні факти (не очевидні)
- **Конект до БД лише через pooler**: `postgresql://postgres.<ref>:<пароль>@aws-0-eu-west-1.pooler.supabase.com:5432/postgres?sslmode=require`. Прямий `db.<ref>.supabase.co` НЕ резолвиться. Рядок у `.env` (`SUPABASE_DB_URL`), у git не йде.
- `patients_best.short_name` — **generated** колонка (не вставляти).
- `lsmd.id_case` — внутрішній послідовний ключ 1…N (НЕ номер картки helsi); номер картки → в `helsi_no`.
- Ідемпотентність імпорту — через `WHERE NOT EXISTS (helsi_no)` (не ON CONFLICT — частковий індекс не інфериться).
- `pypdf` розділяє поля пробілами, `PyPDF2` склеює — парсер толерантний до обох.
- Залежності скрипта: `scripts/requirements.txt` (psycopg[binary], python-dotenv, pypdf).

## Команди
```bash
# імпорт із PDF (з будь-якої машини)
pip install -r scripts/requirements.txt
# .env: SUPABASE_DB_URL=...pooler...
python scripts/import_helsi_cases.py file.pdf            # CSV-звірка
python scripts/import_helsi_cases.py file.pdf --check    # dry-run у БД (відкат)
python scripts/import_helsi_cases.py file.pdf --commit   # запис
```
